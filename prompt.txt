Goal: Fix “markdown feeling” by making Tailwind Typography plugin actually generate prose styles in Tailwind v4.

1) Update src/app/globals.css
Right after:
  @import "tailwindcss";

Add:
  @plugin "@tailwindcss/typography";
  @config "../../tailwind.config.ts";

Notes:
- globals.css is at src/app/, so ../../tailwind.config.ts should point to repo root.
- Keep existing CSS variables and background gradient.

2) Simplify prose overrides (very important)
In globals.css, remove the large block that sets:
  .prose { --tw-prose-body: ... --tw-prose-headings: ... etc }
Because prose-invert already sets dark-mode typography palettes and we want the plugin defaults for spacing + type scale.

Keep ONLY small, safe overrides like:
- link underline behavior
- inline code styling (border + background)
- pre styling (border + radius)
- maybe heading letter spacing if you want

Example target:
@layer base {
  .prose :where(h1, h2, h3) { letter-spacing: -0.02em; }
  .prose a { text-decoration: none; text-underline-offset: 4px; }
  .prose a:hover { text-decoration: underline; }

  .prose code { border: 1px solid var(--border); border-radius: 0.375rem; background: color-mix(in srgb, var(--code) 70%, transparent); padding: 0.15rem 0.4rem; }
  .prose pre { border: 1px solid var(--border); border-radius: 0.75rem; }
}

Remove the custom margin rules you added for p/ul/ol as well — let the typography plugin control spacing.

3) Ensure pages use typography correctly
In each page component, ensure wrapper is:
  <article className="prose prose-invert max-w-none">

4) Verify expected output
After changes:
- h1 should be noticeably larger than paragraph
- h2 should be smaller than h1 but clearly styled
- lists should have proper indentation and spacing
- paragraphs should have consistent vertical rhythm

Deliverables:
- globals.css updated with @plugin and @config
- prose overrides simplified
- app still runs with npm run dev
